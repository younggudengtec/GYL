hexo-githubPage-Blog 配置说明

域名与githubPage对接

域名解析

yg.younggu.xyz 域名的 A 解析 192.30.252.153
这里需要换成自己的域名.
yg.younggu.xyz 域名的 CNAME 解析: youngguyl.github.io.
需要替换自己的域名, github 的子域名也需要修改为自己的.

若有多个子域名,解析的值一样.换掉子域名即可.

e.g:
a.abc.com ‘A’ 192.30.252.153
a.abc.com ‘CNAME’ xxx.github.io. xxx为你自己在github注册的地址.
另外一个子域名, 解析的地址也一样.
b.abc.com ‘A’ 192.30.252.153
b.abc.com ‘CNAME’ xxx.github.io. xxx为你自己在github注册的地址.

github设置

youngguyl/yonglinPage, 进入设置项

  1. GitHub Pages
    1 Source 选择gh-pages branch使用gh-pages branch 作为静态页面
    2 Custom domain, 设置yg.younggu.xyz, 需要和域名解析一致.
  2. 文件需要需要增加一个CNAME的文件. 目的是为了让github知道是哪个子域名
    1 内容为yg.younggu.xyz即可

本地生成 git SSH Key

配置 git

git config --global user.name "youngguyl"
git config --global user.email "7188209@qq.com"

生成 SSH Key

  1. 生成ssh
    ssh-keygen -t rsa -C "7188209@qq.com"

提示密钥文件的地址 /home/pi/.ssh/id_rsa 可以修改

  1. 添加密钥到 ssh-agent
    ssh-agent -s

  2. 添加 SSH key 到 id_rsa
    ssh-add ~/.ssh/id_rsa 生成一个 ~/.ssh/id_rsa.pub文件.

github 新建一个 SSH

settings -> SSH and GPG keys 新建一个SSH,名称容易区分即可把 ~/.ssh/id_rsa.pub的密钥填进去.

测试是否成功

ssh -T git@github.com 如果出现下面的提示, 那就成功了.

配置 hexo,

编译 hexoblog文件夹\_config.yml

上传静态博客

在博客文件夹中, 安装 hexo-deployer-git

npm install hexo-deployer-git --save
不安装, 会出现 error deployer not found:git 错误

上传静态博客

hexo g 生产静态博客
hexo d 上传博客到 git.
